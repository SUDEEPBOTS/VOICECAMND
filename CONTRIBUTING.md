# Contributing to VoiceSraver

First off, thank you for considering contributing to VoiceSraver! It's people like you that make open source such a great community. 

As a SUDEEPBOTS project, we welcome contributions from everyone.

## Where do I go from here?

If you've noticed a bug or have a feature request, open an issue! It's generally best if you get confirmation of your bug or approval for your feature request this way before starting to code.

## Fork & create a branch

If this is something you think you can fix, then fork VoiceSraver and create a branch with a descriptive name.

## Implementation Guidelines

- Make sure to keep the code clean and well-documented.
- Test your features locally in a Telegram Voice Chat before submitting a Pull Request.
- Follow standard Python PEP 8 guidelines.

## Submit your PR

Once you're done, submit a Pull Request. Fill out the provided PR template and wait for review by the SUDEEPBOTS team!
